Background: Hierarchical Reasoning Model (HRM)

The Goal

Supervised learning: Given an input, produce an output. Both have shape \([B, L]\) where \(B\) is batch-size and \(L\) is context-length.

Four Learnable Components

\(f_I(\cdot; \theta_I)\): Input embedding

\(f_L(\cdot; \theta_L)\): Low-level network

\(f_H(\cdot; \theta_H)\): High-level network

\(f_O(\cdot; \theta_O)\): Output head

Network Architecture

Each network is a 4-layer Transformer with:

  • RMSNorm (layer normalization)
  • No bias (for efficiency)
  • Rotary embeddings (for positional encoding)
  • SwiGLU activation function

Total Parameters: 27M across all four components

HRM: Two Networks at Different Frequencies

\(f_L\): Low-Level (High Frequency)

Runs frequently (multiple times per cycle)

Outputs latent feature \(z_H\)

Input: \(z_L + z_H + x\)

\(f_H\): High-Level (Low Frequency)

Runs rarely (once per cycle)

Outputs latent feature \(z_L\)

Input: \(z_L + z_H\)

üí° Key Idea: \(f_L\) processes quickly (high frequency), \(f_H\) integrates slowly (low frequency) ‚Äî inspired by brain hierarchies

HRM Forward Pass Example

\(x \gets f_I(\tilde{x})\) // Embed input
// WITHOUT gradients (4 recursions):
\(z_L \gets f_L(z_L + z_H + x)\)
\(z_L \gets f_L(z_L + z_H + x)\)
\(z_H \gets f_H(z_L + z_H)\)
\(z_L \gets f_L(z_L + z_H + x)\)
\(z_L \gets z_L.\text{detach}()\)
\(z_H \gets z_H.\text{detach}()\)
// WITH gradients (2 recursions):
\(z_L \gets f_L(z_L + z_H + x)\)
\(z_H \gets f_H(z_L + z_H)\)
\(\hat{y} \gets \text{argmax}(f_O(z_H))\) // Predict output

Parameters: \(n=2\) recursions per cycle, \(T=2\) cycles

Total depth: 6 function evaluations (4 + 2)

Deep Supervision: Iterative Improvement

The Concept

Improve the answer through multiple supervision steps while carrying latent features \((z_L, z_H)\) as initialization

How It Works

For each supervision step \(s = 1, ..., N_{sup}\) (up to 16):

  1. Run forward pass through HRM ‚Üí get prediction \(\hat{y}\)
  2. Compute loss between \(\hat{y}\) and ground truth \(y\)
  3. Detach \(z_L\) and \(z_H\) from computation graph
  4. Use detached latents as initialization for next step
  5. Backpropagate and update weights

Why Detach?

Detaching prevents gradients from flowing through previous supervision steps. This provides residual connections without the memory cost of storing all intermediate computations!

Result: Emulates a very deep network (384 effective layers) without requiring backpropagation through all of them!

Fixed-Point Theorem and 1-Step Gradient Approximation

The Assumption

If \((z_L, z_H)\) reaches a fixed-pointA state where applying the function doesn't change the values: f(z*) = z* through recursion:

\(z_L^* \approx f_L(z_L^* + z_H + x)\)
\(z_H^* \approx f_H(z_L + z_H^*)\)

The Theory

Implicit Function Theorem (IFT) + 1-step gradient approximation:

When a recurrent function converges to a fixed point, backpropagation can be applied in a single step at that equilibrium point

The Justification

This theorem justifies backpropagating only through the last 2 of 6 function evaluations (1 \(f_L\) and 1 \(f_H\)), which greatly reduces memory demands

‚ö†Ô∏è Problem: Is a fixed-point actually reached? (We'll discuss this in the next section!)

Adaptive Computational Time (ACT)

The Problem

With deep supervision, each mini-batch requires \(N_{sup}=16\) supervision steps before moving to the next batch. This is very expensive!

The Solution: ACT with Q-Learning

Learn when to halt using a Q-learning objective with two losses:

Halting Loss:

Predicts if current answer is correct

Continue Loss:

Determines if continuing would be better

The Cost

Continue loss requires an extra forward pass through HRM ‚Üí 2 forward passes per optimization step

The Benefit

Average supervision steps during training: < 2 (instead of 16!) on Sudoku-Extreme

At test time: Use full \(N_{sup}=16\) steps for maximum accuracy

HRM Summary: Effective Depth Without Full Backpropagation

The Key Innovation

Deep supervision + 1-step gradient approximation provides a more biologically plausible and computationally efficient alternative to Backpropagation Through Time (BPTT)

Effective Depth Calculation

\(\text{Effective Depth} = n_{\text{layers}} \times (n+1) \times T \times N_{sup}\)

\(= 4 \times (2+1) \times 2 \times 16 = \mathbf{384 \text{ layers}}\)

What HRM Achieves

‚úÖ High accuracy on puzzle tasks

‚úÖ Extremely deep effective network

‚úÖ Memory-efficient training

‚úÖ Early stopping with ACT

üí° But... Is HRM optimal? Are all the complexities necessary? (Next section explores this!)

Problem #1: Fixed-Point Assumptions Are Questionable

The IFT Theorem Requirement

The Implicit Function Theorem with 1-step gradient approximation requires that \((z_L, z_H)\) reaches a fixed-point

The Reality

There is NO guarantee that a fixed-point is reached!

  • Deep equilibrium models normally do fixed-point iteration to solve \(z^* = f(z^*)\)
  • HRM doesn't iterate to convergence ‚Äî just does forward passes
  • After first loop (2 \(f_L\) + 1 \(f_H\)), only 1 more \(f_L\) evaluation before assuming fixed-point!

Evidence from the Paper

HRM authors show residuals with \(n=7\), \(T=7\) (not the \(n=2\), \(T=2\) used in experiments):

  • \(z_H\) residual is clearly well above 0 at every step
  • \(z_L\) residual only becomes closer to 0 after many cycles, but remains significantly above 0
  • \(z_L\) is very far from converged after one \(f_L\) evaluation at \(T\) cycles

üéØ Solution Preview: TRM bypasses this by backpropagating through the full recursion process ‚Äî no fixed-point assumption needed!

Problem #2: ACT Requires Twice the Forward Passes

The ACT Implementation in HRM

ACT uses a Q-learning objective with two losses to decide when to halt:

Halting Loss

Predicts if current answer matches ground truth

‚úÖ Requires 1 forward pass (already done)

Continue Loss

Determines if continuing would be better

‚ùå Requires an extra forward pass!

The Cost

Result: ACT requires 2 forward passes per optimization step

While ACT optimizes time per sample, it doubles the compute per step!

üéØ Solution Preview: TRM removes the continue loss, keeping only the halting loss ‚Äî 1 forward pass is enough!

Problem #3: Complex Biological Arguments

The HRM Justification

HRM justifies two latent variables and hierarchical design based on biological arguments:

  • Different temporal frequencies in the brain
  • Hierarchical processing of sensory inputs
  • Matching to actual brain experiments on mice

The Problems

Too Complex: Biological arguments are very far from artificial neural networks ‚Äî hard to parse what's helping what and why

Unclear Design: Why exactly 2 features (\(z_L\) and \(z_H\))? Why not 1, 3, or more?

Lack of Ablations: No ablation table showing which parts help ‚Äî over-reliance on biological arguments instead

üéØ Solution Preview: TRM provides a much simpler, non-biological reinterpretation ‚Äî and uses just a single network!

Introducing Tiny Recursive Models (TRM)

The Core Philosophy

TRM is MUCH simpler than HRM while achieving significantly better generalization

Key Simplifications

‚úÖ Single Network: One 2-layer network (not two 4-layer networks)

‚úÖ No Fixed-Point: Full backpropagation through recursion

‚úÖ Simpler ACT: Only 1 forward pass (not 2)

‚úÖ Clear Interpretation: No biological arguments needed

Parameters: 7M (TRM) vs 27M (HRM) ‚Äî 74% reduction!

TRM Architecture Overview

Key Components

  • \(x\): Input (problem/question)
  • \(y\): Current solution
  • \(z\): Latent reasoning trace
  • net: Tiny 2-layer network

Process

TRM recursively improves \(y\) through multiple iterations, using \(z\) as internal reasoning

A Simpler Reinterpretation: No Hierarchy Needed!

HRM's View (Complex)

\(z_L\) and \(z_H\) are hierarchical features operating at different temporal frequencies (like the brain)

TRM's View (Simple)

\(y\) (was \(z_H\))

Current Solution

The embedded answer that can be decoded via argmax

üí° This is what we're trying to improve!

\(z\) (was \(z_L\))

Latent Reasoning

Internal "chain-of-thought" that helps improve \(y\)

üí° Cannot be decoded directly!

Why 2 features? We need both: \(y\) to remember the current answer, and \(z\) to track how we got there (reasoning trace)

üéØ No biology needed! This is a natural, intuitive design for iterative reasoning

TRM Forward Pass: Full Recursion with Backprop

Latent Recursion (\(n=6\) steps)

// Recursively improve latent reasoning z:
for i in range(n):
    \(z \gets \text{net}(x, y, z)\) // Update reasoning given question, answer, and current reasoning

// Use reasoning to refine answer:
\(y \gets \text{net}(y, z)\) // Update answer given current answer and reasoning

Deep Recursion (\(T=3\) cycles)

// Run T-1 cycles WITHOUT gradients:
with torch.no_grad():
    for j in range(T-1):
        y, z = latent_recursion(x, y, z, n)

// Run 1 cycle WITH gradients:
y, z = latent_recursion(x, y, z, n)

Key Difference: TRM backpropagates through the full \(n+1\) recursions (not just last 2) ‚Äî no fixed-point assumption!

Ablation: Why Exactly 2 Features?

Experimental Verification

Approach # Features Sudoku Accuracy
TRM (y, z) 2 87.4%
TRM multi-scale z 7 (n+1) 77.6%
TRM single z 1 71.9%

Why More Features (>2) Hurts

Splitting \(z\) into multiple hierarchical features adds complexity without benefit ‚Äî no need for hierarchy!

Why Single Feature (<2) Hurts

Without separate \(y\) and \(z\), the model must store the solution within \(z\) instead of using \(z\) for latent reasoning

‚úÖ Conclusion: 2 features (\(y\) and \(z\)) is the simplest and most effective design!

Innovation #1: Single Network Instead of Two

The Insight

Since \(z \gets f_L(x+y+z)\) contains \(x\) but \(y \gets f_H(y+z)\) doesn't contain \(x\), the task is directly specified by the inputs!

HRM

Two separate 4-layer networks

\(f_L\): iterates on \(z\)

\(f_H\): updates \(y\)

Result: 27M parameters

TRM

One 2-layer network

Same network for both tasks!

Input tells it what to do

Result: 7M parameters

Impact on Sudoku-Extreme: 82.4% ‚Üí 87.4% (single network is better!)

Innovation #2: Less is More ‚Äî 2 Layers > 4 Layers

The Surprising Discovery

Adding more layers decreased generalization due to overfitting!

The Trade-off:

  • Fewer layers: Reduce capacity per step ‚Üí less overfitting
  • More recursions: Increase \(n\) proportionally to maintain compute
  • Result: 2 layers with \(n=6\) beats 4 layers with \(n=3\)

Why This Works

With very small data (~1000 examples), deep recursion with tiny networks prevents overfitting better than large networks!

Impact on Sudoku-Extreme: 79.5% (4 layers) ‚Üí 87.4% (2 layers)

Innovation #3: Attention-Free for Small Fixed Grids

The Observation

Self-attention is great for long sequences (\(L \gg D\)), but for small fixed context (\(L \leq D\)), a simple MLP can work better!

Self-Attention

Parameters: \([D, 3D]\) matrix

Good for: Variable/long sequences

Example: Text, large images

MLP on Sequence

Parameters: \([L, L]\) matrix

Good for: Small fixed grids

Example: 9√ó9 Sudoku (81 positions)

Task-Specific Results

Task Grid Size MLP Attention

üí° Lesson: Match architecture to task constraints ‚Äî not every problem needs attention!

Innovation #4: Exponential Moving Average (EMA)

The Problem

On small datasets (Sudoku-Extreme, Maze-Hard), models tend to overfit quickly and then diverge

The Solution: EMA

Exponential Moving Average of weights with decay = 0.999:

\(\theta_{\text{EMA}} \gets 0.999 \cdot \theta_{\text{EMA}} + 0.001 \cdot \theta\)

Why It Helps

‚úÖ Prevents Collapse: Smooth weight updates avoid sharp divergence

‚úÖ Better Generalization: Averaged weights are more stable

Impact on Sudoku-Extreme: 79.9% (no EMA) ‚Üí 87.4% (with EMA)

üí° Common in: GANs and diffusion models for training stability
